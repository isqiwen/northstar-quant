# 配置说明

## 配置来源优先级

1. 环境变量
2. `.env`
3. `configs/*.yaml`
4. 代码默认值

## 主要配置文件

- `configs/app.yaml`：应用级配置
- `configs/strategy/*.yaml`：策略配置
- `configs/risk/*.yaml`：风控配置
- `configs/data/*.yaml`：数据配置

其中 `configs/app.yaml` 当前也承载日志配置，例如：

- 日志级别
- 是否输出到控制台
- 是否写入日志文件
- 日志目录与文件名
- 按日滚动与保留天数

当前日志系统会把 `command`、`strategy`、`symbol` 等上下文字段直接写入日志记录本身。控制台日志保留 `|` 风格的可读格式，主干顺序为时间、级别、`file:line`、消息；文件日志使用 JSON Lines，字段顺序为 `timestamp`、`level`、`file`、`line`、`msg`，再跟随业务字段。
滚动后的日志文件名采用 `northstar-YYYY-MM-DD.log` 风格，当前活动日志文件保持为 `northstar.log`。

## 设计建议

- 每个策略一个 YAML
- 每个市场一个 universe 配置
- 每个账户一个 execution 配置
