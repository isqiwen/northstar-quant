# 实盘执行现状与增强说明

## 1. 现在实盘是怎么做的

在 v4 之前，Northstar Quant 的实盘主流程是一个**单次执行流程**：

1. 读取最新行情 parquet
2. 运行 ETF 日频轮动 + 动量策略
3. 合成组合目标权重
4. 从 IBKR 同步真实持仓 / 未完成订单 / 近期成交
5. 生成再平衡计划
6. 通过统一路由器提交订单
7. 把订单结果写入数据库
8. 发送企业微信 / Telegram 或控制台告警

对应入口命令：

```bash
northstar live run
```

这个流程已经能完成“从策略到下单”的闭环，但还偏向**一次性触发**，更像手动执行或 cron 触发。

## 2. v4 新增了什么

### 2.1 真实持仓差异分析

新增命令：

```bash
northstar live drift
```

功能：
- 比较目标权重与最新真实持仓权重
- 输出总偏离、最大偏离、每个 ETF 的偏离详情

### 2.2 订单状态轮询 / 成交回写

新增命令：

```bash
northstar live poll
```

功能：
- 从 IBKR 拉取未完成订单状态
- 更新本地订单状态
- 拉取成交并去重写入 fill_records
- 追加同步日志

### 2.3 日频调度器

新增命令：

```bash
northstar live scheduler
```

功能：
- 定时执行 broker sync
- 定时执行 daily rebalance
- 定时生成 daily / weekly / monthly 报告

默认 cron 配置在 `.env` 中通过 `NORTHSTAR_*_CRON` 控制。

## 3. 为什么 Backtrader 还保留

Backtrader 在 Northstar Quant 里承担的是**可信回测层**，不是主研究引擎，也不是实盘执行引擎。

分工如下：

- `vectorbt`：快速研究、参数扫描
- `Backtrader`：更接近真实订单流程的回测验证
- `live + execution`：真实实盘执行

所以，Backtrader 是**使用了的**，但它不负责实盘下单。

## 4. 推荐实盘运行方式

### 开发 / 联调阶段

```bash
northstar sample-data
northstar backtest event etf_rotation
northstar backtest bt etf_rotation
northstar live preview-rebalance
```

### IBKR paper trading 阶段

```bash
northstar live sync
northstar live drift
northstar live run
northstar live poll
```

### 稳定后进入自动调度

```bash
northstar live scheduler
```

## 5. 仍然是骨架，不是最终成品

v4 已经把：
- 真实持仓同步
- 订单状态轮询
- 成交回写
- 日频调度器
- 企业微信 / Telegram 告警
- 日报 / 周报 / 月报

都接入到统一骨架里了。

但它依然属于**个人量化可维护的现代化工程骨架**，不是已经帮你接好全部 IBKR 生产参数和交易细节的最终成品。
你还需要根据自己的账户、交易时间、ETF 池、限价规则进一步联调。
