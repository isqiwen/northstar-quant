# 邮件发送日报、周报、月报

这一版在 v6 的基础上加入了邮件发送能力。

## 一、设计思路

系统现在支持三种通知通道：

- **企业微信 / Telegram**：适合实时告警
- **邮件**：适合发送日报 / 周报 / 月报，并长期归档

报告流程是：

1. 先本地生成 Markdown 报告
2. 再把 Markdown 转成 HTML 邮件
3. 通过 SMTP 发送给一个或多个收件人

## 二、环境变量

在 `.env` 中配置：

```bash
NORTHSTAR_SMTP_HOST=smtp.example.com
NORTHSTAR_SMTP_PORT=465
NORTHSTAR_SMTP_USERNAME=your_user
NORTHSTAR_SMTP_PASSWORD=your_password
NORTHSTAR_SMTP_SENDER=bot@example.com
NORTHSTAR_SMTP_USE_SSL=true
NORTHSTAR_REPORT_RECIPIENTS=you@example.com,backup@example.com
NORTHSTAR_REPORT_EMAIL_SUBJECT_PREFIX=Northstar Quant
```

## 三、命令行用法

### 1. 生成日报并立即发送

```bash
northstar report daily --send-email
```

### 2. 生成周报并立即发送

```bash
northstar report weekly --send-email
```

### 3. 生成月报并立即发送

```bash
northstar report monthly --send-email
```

### 4. 单独发送已经存在的报告

```bash
northstar report send reports/etf_rotation_daily_report.md
```

## 四、调度器行为

当你启动：

```bash
northstar live scheduler
```

到达日报 / 周报 / 月报时间时，系统会：

1. 自动生成对应 Markdown 报告
2. 尝试通过 SMTP 发送邮件
3. 发送成功后再发一条企业微信 / Telegram 信息提示

如果未配置 SMTP，系统只会本地生成报告，不会报错退出。
