# 限价执行、超时撤单、交易日历与 Dashboard

本版本在 v4 基础上新增四个关键能力：

1. **订单超时撤单逻辑**
2. **限价单执行器**
3. **交易日历过滤**
4. **实盘状态 Dashboard**

## 一、订单超时撤单

命令：

```bash
northstar live cancel-stale
```

逻辑：
- 读取本地 `order_records` 中仍处于未完成状态的订单
- 如果 `submitted_at` 已超过 `NORTHSTAR_ORDER_TIMEOUT_SECONDS`
- 则通过 `IBKRBrokerAdapter.cancel_order()` 发起撤单
- 成功后把本地订单状态更新为 `Canceled`

## 二、限价单执行器

默认在 IBKR 实盘下，`live run` 会优先把再平衡计划转成限价单。

当前版本策略：
- 买单：在参考价之上加一定 bps
- 卖单：在参考价之下减一定 bps

这样做的好处是：
- 比市价单更可控
- 对 ETF 日频再平衡通常已经足够

## 三、交易日历过滤

调度器现在会在执行任务前先判断今天是不是交易日。
默认日历：`XSHG`。

如果当天不是交易日，则：
- 跳过同步 / 再平衡任务
- 发送一条说明性告警

## 四、Dashboard

命令：

```bash
northstar dashboard run
```

默认使用 Streamlit，展示：
- 最近真实持仓
- 最近订单
- 最近成交
- 最近报告文件
- 数据库连通性

## 五、建议的日频运行链路

1. 调度器每天在交易日固定时刻执行
2. 先同步券商状态
3. 计算目标组合并生成再平衡计划
4. 发出限价单
5. 周期性轮询订单与成交
6. 对超时订单执行撤单
7. 通过 Dashboard 与告警查看系统状态
