# 模块设计说明

## 模块列表

- `config`：配置加载与校验
- `common`：通用常量与工具函数
- `logging_`：结构化日志
- `db`：数据库模型与会话
- `data`：行情、特征、持久化
- `research`：研究与扫描
- `backtest`：可信回测
- `portfolio`：目标仓位与组合构造
- `risk`：多层风控
- `execution`：订单、适配器、路由
- `monitoring`：告警、健康检查、日报
- `live`：实盘调度入口
- `reporting`：图表与报告

## 包设计原则

- 每个包都尽量只有一个清晰职责
- 核心流程尽量通过 dataclass / pydantic 模型传递
- 不要在 notebook 里写生产逻辑
